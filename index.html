What about team projects?

What does a team project look like?

  • More files and folders
  • Many features to be built
  • Multiple people making contributions
  • A need to experiment and figure things out
  • Mistakes happen - bad assumptions, bugs, confusing code...
  • Need for precise communication

What would go wrong if we worked like this?

													our_project_v1_final2_alice_auth

What could go wrong?

  • Confusing version names => mistakes
  • Hard to consolidate changes to code files if people need to work on the same file. Need for line by line precision
  • People overwrite and break each other's work without meaning to
  • A lot of time gets wasted on rework and on coordinating with each other - very wasteful team dynamic

Without Git, we need to...

  • Spend a lot of time on coordinating dev tasks
  • Spend more time fixing human error (=> personal conflicts)
  • Hire another project manager just to juggle all this

But we have the technology to solve all these problems!

Git downsides

  • Hard to explain to non-technical stakeholders + noobs
  • There is a learning curve. It's weird at first
  • Many people gloss over Git's super-power features because they want quick wins and confidence
